LLM chaos testing, minus the ceremony

Break the provider.
Not your app.

AgentBreak is a tiny proxy that injects slowdowns, 429s, 500s, and flaky responses into OpenAI-compatible traffic so you can see how your app behaves before production does it for you.

Install from PyPI View source
Mock or proxy Run fake responses or forward real traffic upstream.
Fast setup Point `OPENAI_BASE_URL` at localhost and start testing. 
pip install agentbreak

agentbreak start \
  --mode mock \
  --scenario mixed-transient \
  --fail-rate 0.2

export OPENAI_BASE_URL=http://localhost:5000/v1
export OPENAI_API_KEY=dummy

Enough to validate retries, fallbacks, timeout handling, and duplicate request behavior locally.

How it works

Sit AgentBreak between your app and an OpenAI-compatible endpoint. Then inject controlled failure while keeping the interface familiar.

your app -> AgentBreak -> provider or mock

Useful endpoints

Track what happened while your client was retrying or failing. 

curl http://localhost:5000/_agentbreak/scorecard
curl http://localhost:5000/_agentbreak/requests

Minimal by design

It only covers the parts you need to pressure-test app resilience.

  • Failure injection: `429`, `500`, `503`, auth errors, and latency.
  • Traffic modes: run fully mocked or proxy real upstream calls.
  • Inspection: inspect requests and review a small scorecard.
  • Target: `POST /v1/chat/completions` for OpenAI-compatible apps.
